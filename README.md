处理用户点击日志，从点击日志的用户中，随机抽取5万用户作为验证集用户。把验证集用户的最后一次点击记录从点击日志中删除，加入到查询日志，用作模型效果验证

# 2. 多路召回

- **ItemCF**
    - 基于物品的协同过滤召回，根据用户点击历史序列中的文章，召回与历史文章相似的文章。
    - 权重衰减策略：根据文章在历史序列中位置的先后，以及点击时间距离最后一次点击时间的插值，进行相应的权重衰减。
    - 物品相似度计算方式：出现在同一用户点击历史序列中的文章，认为是相似的，根据两个文章间隔位置的远近、前后关系以及点击时间间隔，计算物品相似度。

- **UserCF**
    - 基于用户的协同过滤召回，根据用户向量，找到与查询用户相似的用户，返回相似用户历史行为序列中查询用户未看过的文章。
    - 用户向量表示：利用用户点击历史文章的内容向量计算用户向量，并根据文章在历史序列中的位置以及时间戳，对文章向量进行加权，得到用户向量。
    - 物品召回分数计算：针对相似用户历史行为序列中的文章，利用相似用户的相似度计算物品召回分数，根据文章在历史序列中的位置以及时间戳对相似度加权，得到物品召回分数。

- **二部图召回**
    - 基于二部图网络的召回，将用户和文章视为二部图的顶点，根据文章顶点的相同邻居的数量关系，即相同点击用户的数量确定文章间的相似度关系。
    - 权重衰减策略：针对热门物品的惩罚，物品被越多的用户点击过，则其相应的权重就会越低，从而减少热门物品在召回结果中的影响。

- **item2vec召回**
    - item2vec：将文章id根据用户进行分组，形成“句子”集合，即用户点击历史序列。利用word2vec模型进行训练，将每个文章id映射为一个向量。
    - 相似物品召回：根据用户最近交互过的物品向量，利用相似最近邻搜索，召回一批物品向量，利用2-向量间距离作为物品相似度，并结合序列位置衰减和时间衰减策略，最后返回相似度最高的K个物品

- **热门物品召回**
    - 热门物品召回，根据文章被点击的次数，对文章进行排序，返回点击次数最多的前K个文章。
    - 权重衰减策略：在统计点击次数的基础上，根据点击时间距离最后一次点击的远近，点击用户的多样性以及用户的长短期类别兴趣爱好进行加权。

- **冷启动**
    - 针对冷门物品的召回，认为总点击次数少于K次的文章为冷门物品，用于后续筛选。
    - 冷门物品筛选策略：根据点击次数初步筛选出候选物品，在通过人为设定的规则进行进一步筛选，包括新闻类别，文章字数，创建时间。

- **多路召回合并**
    - 加权融合：通过人为设定的权重对相似度进行加权计算，每个用户保留分数最高的前topk个文章
    - 逻辑回归融合：通过训练一个简单的逻辑回归模型，自动地学习不同召回路径的权重。

# 3. 特征工程

- **用户特征**
    - 用户id
    - 用户最后一次点击系统
    - 用户最后一次点击环境
    - 用户最后一次点击设备组
    - 用户最后一次点击国家
    - 等等

- **新闻特征**
    - 文章id
    - 所属类别id
    - 创建时间
    - 文章字数
    - 等等

- **用户文章交互特征**
    - 文章的创建时间与用户最后一次点击的时间差
    - 文章字数与用户最后一次点击的文章字数差
    - 文章类别在用户交互历史中出现的次数
    - 等等

# 4. 排序
- 对召回后的结果进行排序，采用了三种不同的排序模型，分别为：
    - LGB分类模型
    - LGB排序模型
    - DIN模型
- 实现了两种模型融合方式，对模型结果进行加权融合，及通过训练一个简单的逻辑回归模型对各个模型的结果进行集成
